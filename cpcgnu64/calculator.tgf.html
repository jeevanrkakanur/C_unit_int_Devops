Control Coupling Report

with Function as Module

Show/hide documentation

This report displays in the Summary table the number of Control Couplings of your application, given the information you provided and the percentage of Control Couplings that have been covered, i.e. executed by at least one test case. It contains also 2 charts showing the percentage of covered Control Couplings for all the application and per pair of compilation unit.

Filters can be applied: a selection of calling compilation units and a selection of called compilation units. Filters can also be applied to 'Details' and 'Call Graph' sections. A switch can show or hide all the graphs to have more compact report.

The Details part of the report groups the Control Couplings by pair of compilation units and displays for each of them the calling compilation units, the control flow, the list of test cases that covered this control flow, the associated requirements and, if this control coupling has been covered. If the .tgf/tio files are generated using unit testing, the test cases are the tests in the .ptu files identified by the service name and the test name and the requirements are those set by the keyword REQUIREMENT in these tests. If not, the test cases are displayed with their execution date. At the end of each group, a Call Graph displays all the functions of the calling and called compilation units as an interactive call graph: Select a control coupling in the table to highlight it in the call graph.
