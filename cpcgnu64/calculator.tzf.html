Worst Stack Size Report

Show/hide documentation

This report displays in the Summary table an evaluation of the worst stack size of your application. If you provided the list of entry points, it is computed by entry point, as well as all the following metrics. It shows also the number of control flows, the number of control flows for which the stack size cannot be computed because of missing function stack sizes (in red if positive), and the number of recursive control flows that also prevent the report from providing a correct estimation of the worst stack size (in red if positive). The 2 last lines are the number of functions, and functions without stack information (in red if positive).

If you provided the list of entry points, the maximum stack size allowed and, optionally, a percentage of available stack size for security, the report displays also total stack size and verify if this size does not go over the available stack size. All this information are also gathered in a chart.

The Details part of the report displays the first control flows sorted by decreasing stack size. On top of the table, you can choose the number of control flows to display between 10 to all. You can also select an entry point (available only if you provided the list of entry points). In such case, the details and functions tables and the graph are updated to show only the control flows and the functions for this entry point. The last switch button allows to show/hide the final graph.

The Functions part of the report displays all the functions found in the information you provided with their relative stack size. The ones without stack size are displayed in red.

The Call Graph part displays all the functions as an interactive call graph from left to right or from top to bottom. Select a control flow in the table to highlight it in the call graph.
