Data Coupling Report

Show/hide documentation

This report displays in the Summary table the number of Global Variables found in your application and the number of Data Couplings relative to these global variables. A Data Coupling is a pair of def/use (a def is a write of a global variable, a use is a read of a global variable). Then the table gives the number and the list of the global variables that have no Data Coupling, i.e. that are read but never written, or are written but never read. The last line of the table gives the percentage of Data Coupling that are covered (a Data Coupling is covered if the associated def/user pair has both def and use executed in at least one test). The following charts summarize this information: the first one for the global coverage of Data Couplings, the second one for the number of Data Couplings covered or not per global variables.

The Details part of the report groups the def/use pairs by Data Couplings and displays for each of them the list of test cases that covered this def/use pair, the associated requirements and if this data coupling has been covered. If the .tio files are generated using unit testing, the test cases are the tests in the .ptu files identified by the service name and the test name and the requirements are those set by the keyword REQUIREMENT in these tests. If not, the test cases are displayed with their execution date.

At the end of the report, a Call Graph displays all the global variables with their interactions with the functions of the application that read or/and write them. The arrows between them represent a def or a use (depending of the sense of the arrow). It is green if the corresponding def or use has been covered.
